# Suwol Visual Reference v0.2.4

This patch fixes the Release workflow manual dispatch bug where the selected workflow ref could be treated as the release tag.

## Fixed

- Release workflow manual runs now read `github.event.inputs.release_tag`.
- Manual Release workflow runs reject branch names such as `main` and require a version tag like `v0.2.4`.
- Windows/Linux release artifacts remain on the current v0.2.x distribution policy: Windows ZIP plus Linux AppImage and `latest-linux.yml`.
- GPG signing remains aligned with the SuwolView release pattern, including temporary `GNUPGHOME` and organization secrets.
- macOS attach remains separate and can attach signed/notarized macOS artifacts to the same release after the core release is published.

## Expected Core Assets

- `SuwolVisualReference-0.2.4-win-x64.zip`
- `SuwolVisualReference-0.2.4-linux-x64.AppImage`
- `latest-linux.yml`
- `checksums.txt`
- `checksums.txt.asc`
- `SuwolVisualReference-0.2.4-checksums.txt`
- `SuwolVisualReference-0.2.4-checksums.txt.asc`
- `suwol-release-public-key.asc`

## Manual Recovery

If a tag-triggered release needs to be rerun after a workflow-only fix, run the Release workflow from `main` and set `release_tag` to the existing tag.
