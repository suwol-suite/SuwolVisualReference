# Suwol Visual Reference v0.2.5

This patch release consolidates Windows, Linux, and macOS publishing into one Release workflow.

## Fixed

- The Release workflow now builds Windows ZIP, Linux ZIP, Linux AppImage, macOS arm64 DMG, and macOS arm64 ZIP artifacts before publishing.
- macOS artifacts are discovered from the downloaded artifact folder and normalized to stable release names instead of relying on a brittle fixed glob.
- `latest-linux.yml` and `latest-mac.yml` are included in the same checksum and GPG signature set as the app artifacts.
- Release verification now requires every expected Windows, Linux, and macOS asset when the workflow publishes a GitHub Release.
- The Linux AppImage update metadata remains available through `latest-linux.yml`.

## Expected Assets

- `SuwolVisualReference-0.2.5-win-x64.zip`
- `SuwolVisualReference-0.2.5-linux-x64.zip`
- `SuwolVisualReference-0.2.5-linux-x64.AppImage`
- `latest-linux.yml`
- `SuwolVisualReference-0.2.5-mac-arm64.dmg`
- `SuwolVisualReference-0.2.5-mac-arm64.zip`
- `latest-mac.yml`
- `checksums.txt`
- `checksums.txt.asc`
- `SuwolVisualReference-0.2.5-checksums.txt`
- `SuwolVisualReference-0.2.5-checksums.txt.asc`
- `suwol-release-public-key.asc`

## Verification

```bash
gpg --import suwol-release-public-key.asc
gpg --verify checksums.txt.asc checksums.txt
shasum -a 256 -c checksums.txt
```

macOS notarization can take longer than the Windows and Linux packaging jobs. The Release workflow does not publish v0.2.5 until the macOS job and complete asset verification pass.
