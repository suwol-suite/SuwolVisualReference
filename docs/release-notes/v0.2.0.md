# Suwol Visual Reference v0.2.0

Suwol Visual Reference v0.2.0 is the first feature release after the public release tooling pass. It focuses on larger-library browsing, organization, richer media handling, color-based discovery, custom Codex exports, and Linux AppImage update support.

## Highlights

- Dense list view with configurable columns, sorting, filtering, and desktop selection behavior.
- Collection management upgrades, including cover images and manual collection asset ordering.
- Smart folder management with editable multi-condition rules and result count previews.
- GIF, SVG, and video-aware import handling with safer previews and placeholder fallback behavior.
- Palette color search with swatch-to-filter interaction.
- Custom Codex Markdown export template editor with preview and placeholder warnings.
- Linux AppImage release artifact and automatic update metadata support.

## New Features

- Switch between thumbnail grid and dense list views.
- Show, hide, reorder, and resize list columns with saved preferences.
- Sort assets by imported date, name, size, resolution, rating, extension, and collection order.
- Filter assets by extension, rating, tags, aspect, minimum dimensions, memo, source URL, recency, favorites, duplicates, deleted state, and palette color.
- Create, edit, delete, color, describe, and cover collections from the collection manager.
- Reorder assets inside collections with drag handles and move controls.
- Create and edit smart folders with multi-condition rules, validation, and result previews.
- Play animated GIFs in the large viewer while keeping grid and list thumbnails static.
- Use custom Codex export templates with section ordering, placeholder insertion, preview, and unknown-placeholder warnings.
- Open Help/About content for shortcuts, repository, license, third-party notices, known issues, and optional FFmpeg policy.

## Improvements

- SVG previews are rasterized instead of inlined directly.
- Video thumbnail extraction is attempted only through optional external FFmpeg/ffprobe tooling.
- Missing or failed video thumbnail extraction records warnings and falls back to placeholders without failing imports.
- Toolbar and status bar copy now reflect active sorting and filtering state.
- Smoke and selection regression coverage now exercise v0.2 media, color search, export templates, organization queries, collection ordering, and smart folder workflows.
- Release validation now checks ZIP, AppImage, update metadata, checksums, and forbidden release asset paths.

## Linux AppImage Update Support

Linux AppImage builds are the only v0.2.0 artifact type with automatic update support. The AppImage uses GitHub Release metadata from `latest-linux.yml`.

Windows ZIP and Linux ZIP builds remain manual-update distributions. macOS distribution and update support are planned for a later milestone.

The actual update-available flow can only be fully validated after a newer release, such as v0.2.1, exists on GitHub Releases.

## Downloads

- `SuwolVisualReference-0.2.0-win-x64.zip`
- `SuwolVisualReference-0.2.0-linux-x64.zip`
- `SuwolVisualReference-0.2.0-linux-x64.AppImage`
- `latest-linux.yml`
- `SuwolVisualReference-0.2.0-checksums.txt`

Linux desktop environments may require adding execute permission before running the AppImage:

```bash
chmod +x SuwolVisualReference-0.2.0-linux-x64.AppImage
```

## Checksums

Download the checksum file from the same GitHub Release as the app artifact. If signatures are published, verify the detached signature first:

```bash
gpg --import suwol-release-public-key.asc
gpg --verify SuwolVisualReference-0.2.0-checksums.txt.asc SuwolVisualReference-0.2.0-checksums.txt
shasum -a 256 -c SuwolVisualReference-0.2.0-checksums.txt
```

The checksum file should include the Windows ZIP, Linux ZIP, Linux AppImage, and `latest-linux.yml`.

## Known Issues

- Windows builds are unsigned and may trigger SmartScreen warnings.
- Windows ZIP and Linux ZIP builds do not support automatic updates.
- FFmpeg is not bundled. Video thumbnails may show placeholders when external FFmpeg/ffprobe is unavailable.
- macOS builds are not distributed yet.

## Upgrade Notes

- Existing local libraries remain portable folders and do not need manual migration.
- Keep backups of important libraries before using any new release on production reference sets.
- For Linux AppImage update testing, install or run v0.2.0 first, then validate update detection after a later release is published.
