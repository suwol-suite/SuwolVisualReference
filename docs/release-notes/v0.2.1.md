# Suwol Visual Reference v0.2.1

v0.2.1 fixes the Linux packaged app verification failure that occurred during the v0.2.0 release workflow.

## Purpose

The v0.2.0 Linux artifact build completed, but the release workflow failed while verifying the packaged Linux app on a GitHub Actions Ubuntu runner. Electron aborted because the packaged `chrome-sandbox` helper was not configured with the SUID permissions expected by Chromium.

This patch changes the verification command only. It does not change the main v0.2.0 app feature set.

## Changes

- Linux packaged app verifier now runs the packaged Electron app with `--no-sandbox`.
- Linux packaged app verifier applies `ELECTRON_DISABLE_SANDBOX=1` during verification.
- Failure logs now include target platform, executable path, run args, exit code, and stderr/stdout snippets.
- No major app feature changes from v0.2.0.
- v0.2.1 is the recommended release over v0.2.0.

## Expected Assets

- `SuwolVisualReference-0.2.1-win-x64.zip`
- `SuwolVisualReference-0.2.1-linux-x64.zip`
- `SuwolVisualReference-0.2.1-linux-x64.AppImage`
- `latest-linux.yml`
- `SuwolVisualReference-0.2.1-checksums.txt`

## Notes

- Linux AppImage supports automatic update checks through `latest-linux.yml`.
- Windows ZIP and Linux ZIP remain manual update packages.
- Actual update detection requires a future release after installing an older AppImage.
- v0.2.0 tags and failed workflow runs are left intact.
